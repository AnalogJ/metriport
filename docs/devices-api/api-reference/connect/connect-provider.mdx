---
title: "Connect Provider"
description: "Connects the user to a provider"
api: "GET /connect/:provider"
---

## Path Params

<ParamField path="provider" type="string" required>
  A healthcare data provider. See the list of all supported providers
  [here](/devices-api/more-info/our-integrations).
</ParamField>

## Query Params

<ParamField query="state" type="string" required>
  The user's [connect token](/devices-api/api-reference/user/create-connect-token).
</ParamField>

<ParamField query="deviceIds" type="string">
  A comma-separated string containing device ID(s).
</ParamField>
<Info>Example of connecting two devices at once: `device_id=device_id_0001,device_id_1234`.</Info>

<ParamField query="deviceUserId" type="string">
  The user ID (patient ID, with some providers) of the user that is connected to the device.
</ParamField>

<Info>
  The `deviceIds` and `deviceUserId` query parameters should only be used to connect RPM device
  providers, which currently only include [Tenovi](/devices-api/getting-started/tenovi).
</Info>
<Warning>`deviceIds` and `deviceUserId` are **required** for Tenovi.</Warning>

<ResponseExample>

```javascript Metriport SDK
import { MetriportDevicesApi } from "@metriport/api-sdk";

const metriportClient = new MetriportDevicesApi("YOUR_API_KEY");

const token = await metriportClient.getConnectToken("metriportUserId");

await metriportClient.connectProvider("tenovi", token, "device_id_0001", "85300");
```

</ResponseExample>
